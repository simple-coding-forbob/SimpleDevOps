# 클라우드 서비스(Cloud Service) : 구글 드라이브같이 인터넷에서 일정 공간을 대여하고 요금을 받는 서비스
# 구글드라이브 : 인터넷에서 하드디스크 공간 대여
# cpu 대여, 메모리 대여, 스토리지(하드디스크 집합) 대여
# , s/w 대여 (윈도우, 우분투, 레드헷, 한글, ms-word 등)
# 예) AWS(Amazon Cloud Web Service) 첫시작
#    , OCI(오라클 클라우드), GCP(구글 클라우드)
#    , Azure(마이크로소프트 클라우드) 등

# 데이터센터 : 컴퓨터 자원(cpu, 메모리, 스토리지 등)을 하나의 건물에 모아놓고 구축해 놓은 단지(아파트 등)
# 돈 : 5천억 (1조)

# 아마존에서 생성 : (나라별로 지역별로) 대한민국 데이터센터 2개 생성(대여)
# 리전 : 나라별로 관리하는 단위
# 리전(나라(지역) 선택) : 미국 버지니어
# 실습예제에 따라 요금이 대략 약 몇십원 ~ 몇백원 지불될 수 있음
# 해커들 아마존 회원가입 한사람을 타겟으로 해킹을 시도하는 사례가 많다고 합니다.
# 비트코인 채굴(해커) -> 아마존 사례

# 1) aws 개요
# AWS : Amazon web service 의 약자, 아마존에서 제공하는 클라우드 서비스
# 비용을 받고 it 자원을(cpu, 메모리, 스토리지 등) 제공
# 365일 무중단 서비스를 보장(보증)
# 장점 : 필요한 만큼 컴퓨팅 파워를 사용할 수 있고, 유연하게 줄였다가 늘였다가 할 수 있음
# aws 기본 : 리전부터(나라별 데이터센터 선택) 선택
# 레드오션 : 경쟁이 치열한 분야
# 블루오션 : 경쟁이 치열하지 않은 분야
# 리전(나라)마다 비용청구가 다름 : 서울이 미국 버지니아보다 비쌈
# 1년 무료 서비스: 신규 회원가입자에 한해(프리티어)

# 2) aws 계정 만들기
#  - OTP : 1회용 비밀번호 생성기(2중보안)
#  - ROOT 계정 : 관리자용 계정(신규 회원 가입하면 생성됨)
#  - 로그인 : 메인화면에서 우측 상단에 콘솔에 로그인 버튼 클릭

# IAM 계정  : 개발자용 계정
# IAM : 루트(슈퍼 관리자용 계정)이 아니라 일반 사용자 계정으로 몇가지 기능만으로 제한되는 사용자
# 개발 : IAM 계정을 이용해서 개발하는 것을 추천

# 권한 설정 비교
# AWS         |   오라클 DB
# 유저(User)  |   유저(User)
# 그룹(Group) |    
# 역할(Role)  | 권한들의 모임(집합), Connect, Resource
# 정책(Policy)| 권한
# 역할 > 정책 > 권한

# 3) 루트 유저 로그인 및 프리티어 알림 설정
# (1) 루트유저로 로그인
# (2) aws 프리티어 요금이 80%가 넘어가면 이메일로 알려주는 서비스 
#    * 메뉴 선택
#        결제대시보드 - 기본설정 - 청구서 설정 :
#        - 알림 기본 설정 - 편집 버튼 클릭 - aws 프리티어 알림 체크 - 업데이트 버튼 클릭 

# 4) OTP 보안 (2중보안) 설정하기 : 루트 사용자
#  OTP 보안 == MFA 멀티 팩터 인증
#  (1) aws 메인화면에서 검색창에서 IAM 키워드 넣고 검색
#      - IAM 창 왼쪽에서 IAM 에서 다룰 수 있는 사용자, 그룹, 역할, 정책에 대한 메뉴을 확인할 수 있음
#  (2) 보안 권장 사항 : 
#      - 루트 사용자에 대해 MFA 추가 
#        a) MFA 추가 버튼 클릭 
#        b) MFA 할당 버튼 클릭
#        c) MFA 디바이스 이름 : authy
#           MFA 디바이스 선텍 : 인증 관리자 앱 (핸드폰 그림)
#        d) 디바이스 설정 
#           - 핸드폰 앱    : authy 설치 (이메일, 핸드폰 번호 입력)
#           - QR 코드 표시 : authy 앱으로 스캔
#           - MFA 코드 1 : 숫자 6개 (20초 기다림)
#           - MFA 코드 2 : 숫자 6개 

# 5) IAM 유저 만들기
#  (1) 사용자 버튼 클릭 : 유저 목록 화면이 나옴
#  (2) 사용자 추가 버튼 클릭 : 선택 사항 체크 안함
#      - 사용자이름          : scott
#      - aws management console 에 대한 액세스 권한 제공 - 선택 사항 : 체크
#      - IAM 사용자를 생성하고 싶음 : 체크 (직접 암호 관리)
#        a) 사용자 지정 암호 : 자신의 암호 넣기
#        b) 사용자는 다음 로그인 시 새 암호를 생성해야 합니다(권장) : 체크 해제
#  참고) aws management console 액세스 : 콘솔 웹화면에서 비밀번호를 통해 aws 메인 화면에 로그인(웹 브라우저 방식)
#       프로그래밍 방식 액세스            : 엑세스 키와 비밀 키가 제공되며, api 함수를 통해 다양한 aws 리소스의 사용이 가능(코딩 방식)
#  (3) 권한 설정 : 다음 버튼 클릭
#      - 그룹에 사용자 추가 : 기본 선택 
#  (4) 검토 및 생성 : 다음 버튼 클릭
#      - 태그 : 만들지 않음
#  참고) 태그 : 태그명을 넣어 두면 검색하는데 도움이 됨

#  (5) 암호 검색
#      - .csv 파일 다운로드 버튼 클릭 : 
#         a) scott_credentials.csv : 자신의 암호 및 접속 url 확인

#  (6) IAM 유저 삭제하기 : 루트 계정으로 가능
#      - 삭제 버튼 클릭 : 유저 삭제됨
