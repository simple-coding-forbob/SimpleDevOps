# 02_ec2/01_ec2.md
# 3장 EC2
# EC2(Elastic Cloud Computer) : 유연한 클라우드내 가상의 서버(업무용 컴퓨터)
# cpu, 메모리, 네트워크 등을 상황에 맞게 유연하게 늘였다가 줄이기가 가능함

# 1) 비용 지불
#   - 온디맨드 : 시간당 정해진 금액을 지불하는 방식
#         (짧은 시간 동안 임대해서 사용하는 방식)
#   - 리저브드 : 1 ~ 3년 정도 싸게 임대해서 사용하는 방식
#         (장점: 비용이 쌈, 처음 요구했던 용량 그대로 사용해야함)

# 2) EBS : Elastic Block Store, 스토리지(저장장치) 볼륨
#    컴퓨터와 독립적으로 동작, 컴퓨터 전원이 꺼져도 데이터는 보관됨
#  - SSD : 속도 빠름, 비용비쌈, gp2(보통성능), io1(고성능)
#  - HDD : 가장 싼 디스크, 속도 느림, 데이터 백업, 중요하지 않은 데이터 위주로 사용

# 3) ELB : Elastic Load Balancer, 로드밸런서 , 외부에 서비스 노출, 
#         부하분산(load-balancing)

#    - 이름        : aws_was
#    - 애플리케이션 및 OS 이미지(Amazon Machine Image) : Ubuntu 선택
#    - 인스턴스 유형  : t2.micro
#    - 키 페어      : 새 키페어 생성 버튼 클릭
#      a) 키 페어이름 : aws_keypair

#      c) 프라이빗 키 파일 형식
#         - .pem (Mac   유저용) : 선택
#         - .ppk (윈도우 유저용) 
#         - pc 에 aws_keypair.pem 파일 다운로드 되었는지 확인
#    - 네트워크 설정
#      a) 보안그룹 생성 : 기본 체크
#      b) 인터넷에서 SSH 트래픽 허용 : 기본 체크
#      c) 인터넷에서 HTTPS 트래픽 허용 : 체크
#      d) 인터넷에서 HTTP 트래픽 허용 : 체크
#      e) 편집 버튼 클릭
#         - 보안 그룹 규칙 추가 버튼 클릭
#           가. 포트 범위 : 3000
#           나. 소스 유형 : 위치 무관 변경
#           다. 설명     : react#
#      f) 편집 버튼 클릭
#         - 보안 그룹 규칙 추가 버튼 클릭
#           가. 포트 범위 : 8000
#           나. 소스 유형 : 위치 무관 변경
#           다. 설명     : springboot

#     - 스토리지 구성 : 기본 선택

#   (3) 성공 : 모든 인스턴스 보기 버튼 클릭

# 예제 2) EC2 인스턴스에 접속
#   (1) 다운로드 받은 aws_keypair.pem(aws_keypair.ppk) 파일이 있는 위치로 이동 -> 터미널 열기
#    퍼미션 변경 : chmod 400 키파일
#     chmod 400 aws_keypair.pem(aws_keypair.ppk)
#    (2) EC2 서버 정보 확인 : 
#       - EC2 인스턴스 클릭
#       - 연결 버튼 클릭 
#         a) 유저명        : ubuntu
#         b) 퍼블릭 IP 주소 : 18.212.198.87
#    (2) pc 에서 터미널 접속
#       - 맥 : temius 추천
#         a) NEW HOST 버튼 클릭
#            - IP   : 18.212.198.87
#            - 유저명 : ubuntu
#            - set a key 버튼 클릭
#            - 파일을 디래그해서 빈 곳에 넣고 화면 닫기
#            - 메인 화면에 새로 생긴 아이콘이 있고 ip 주소가 보임 : 더블 클릭 시 접속됨
#       - 윈도우 : putty 다운로드 
#            - 유저명 : ubuntu@18.212.198.87
#            - 왼쪽 트리) SSH - Auth -Credential - Public-key authentication private key file for authentication 입력창에 aws_keypair.ppk 파일 경로 
#            - Open 버튼 클릭
