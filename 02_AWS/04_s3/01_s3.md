# S3
# S3  : Simple Storage Service, Aws 의 메인 서비스중의 하나

# 용도 : 주로 이미지 등의 업로드를 위한 스토리지로 값싼 디스크를 사용함, 구글 드라이브 처럼 이미지, 비디오, 압축파일 등 다양한 포맷을 보관 가능
# 장점 : 편리한 업로드 화면으로 구성됨
# 업로드 최대 용량 : 5TB

# (1) S3 의 버킷(Bucket, 물동이)
# 버킷 : 기본적으로 만들어야 하는 것으로 최상위 폴더와 같음
# 버킷명 : 모든 리전(세계, 나라)에서 고유한 이름으로 지정해야함
# 버킷의 구조 : 자바스크립트의 객체형태와 같음(키, 값)
#  - 키 : 파일명
#  - 값 : 파일의 대략적인 제목(title)

# (2) 비용 청구
#  - 버킷(최상위 폴더) 만든다고 비용이 청구되지 않고, 파일 업로드시 GB 당 비용이 청구됨
#  - 저장장치(스토리지) 유형에 따라 추가 비용도 발생할 수 있음

# (3) 버킷 보안
#  - 최초 생성시 기본값으로 만든 사용자만 모든 권한을 가짐
#  - 외부에서는 접근 못함(최초 생성시), 접근할 수 있게 권한 부여가 필요함

# 예제 1) S3 를 만들고 이미지를 업로드 하세요.
# 그리고, 외부에서 접근 가능하도록 권한을 부여하세요

# (1) 버킷 만들기
#   - 일반 구성
#     a) 리전(지역) : 글로벌(지역없음)
#     b) 버킷 버튼 클릭 또는 왼쪽) 버킷 클릭
#     c) 버킷 이름 : aws-forbob-bucket   (이름이 유일해야함)
#     d) AWS 리전  : 미국-동부-(버지니아동부)
#   - 객체 소유권
#     a) ACL 비활성화됨(권장)
#   (참고) ACL : Access Control List, 엑세스 제어 목록
#   - 이 버킷의 퍼블릭 액세스 차단 설정
#     a) 모든 퍼블릭 액세스 차단
#   - 기본 암호화
#     a) 암호화키 유형
#        - Amazon S3 관리형 키(SSE-S3) : 기본 선택
#     b) 버킷 키
#        - 비활성화 체크

# (2) 버킷 - 폴더 만들기
#   - 폴더 만들기 버튼 클릭
#     a) 폴더이름 : image
#     b) 폴더만들기 버튼 클릭
#   - image 클릭
#   - 업로드 버튼 클릭
#     a) 파일 추가 버튼 클릭 : socks_green.jpg 업로드
#     b) 권한 : 기본 선택 유지
#     C) 속성 : 기본 선택 유지(standard)
#  (참고) standard : 스토리지클래스 유형 (기본선택함)

# (3) 권한 설정
#   - aws-forbob-bucket 클릭
#   - 권한 탭 클릭
#   - 퍼블릭 액세스 차단 편집(버킷 설정) 편집 버튼 클릭
#     a) 퍼블릭 액세스 차단(버킷 설정) 체크 해제 
#     b) 변경 사항 저장 버튼 클릭
#   - 객체 소유권 편집
#     a) 객체 소유권
#        - ACL 활성화됨 선택
#        - ACL이 복원된다는 것을 확인합니다. 체크
#        - 객체 소유권 : 버킷 소유자 선호 선택
#   - socks_green.jpg 선택
#     a) 권한 탭 클릭
#     b) ACL(엑세스 제어 목록) : 편집 버튼 클릭
#       - 모든 사람(퍼블릭 액세스) : 읽기(객체) 선택
#       - 이러한 변경 사항이 객체와 버킷에 미치는 영향을 이해합니다. : 선택
#       - 변경 사항 저장 버튼 클릭
